# To-Do

## Назначение
- Нативное iOS‑приложение для управления личными задачами с локальным хранением и синхронизацией с Core Data.
- При первом запуске импортирует данные с `https://dummyjson.com/todos`, после чего работает офлайн.

## Основные возможности
- Список задач с заголовком, описанием, датой создания и статусом выполнения.
- Создание, редактирование, удаление, поиск и переключение статуса без блокировки интерфейса (операции выполняются в фоновых потоках).
- Экран редактирования в формате модального окна с валидацией и индикацией загрузки.

## Архитектура и технологии
- VIPER‑модули `TodoList` и `TodoEditor` обеспечивают изоляцию View/Interactor/Presenter/Entity/Router.
- Репозиторий объединяет Core Data и загрузку через `URLSession`, поддерживает безопасный доступ из фоновых контекстов.
- Доменные модели отделены от DTO и NSManagedObject для облегчения тестирования.

## Системные требования
- macOS 15.7+ и Xcode 16.1 (Swift 5.10).
- iOS Simulator 18.0 или новее (проект проверен на iPhone 17 Pro, iOS 26.1).

## Сборка и запуск
1. Клонируйте репозиторий и откройте `To-Do.xcodeproj`.
2. Выберите схему `To-Do` и симулятор iOS 18.0+.
3. Запустите приложение (`⌘R`). При первом старте произойдёт импорт исходных задач.

## Автоматические проверки
- Ruby‑скрипт `scripts/setup.rb` выполняет проверку окружения, подбирает доступный симулятор и запускает ключевые модульные тесты репозитория.
- Полный прогон модульных тестов:  
  `xcodebuild test -project To-Do.xcodeproj -scheme "To-Do" -destination 'platform=iOS Simulator,name=<device>,OS=<version>'`
- UI‑тесты находятся в таргете `To-DoUITests` и проверяют успешный запуск приложения; при необходимости сценарии можно расширить.

## Структура проекта
- `Application/SceneDelegate` — точка входа и сборка стартового модуля.
- `Data/` — Core Data стек, репозиторий, сетевой слой и сопутствующие модели.
- `Domain/` — доменные модели без зависимостей от UI и хранилища.
- `Modules/TodoList` и `Modules/TodoEditor` — реализуют соответствующие VIPER‑компоненты.
- `Tests/` — модульные тесты для репозитория, интерактора редактора и презентера списка; `UITests/` включает smoke‑сценарий запуска.

## Дополнительные заметки
- Комментарии в коде лаконичны и описывают назначение ключевых классов и методов.
- Все фоновые операции используют `performBackgroundTask`, чтобы UI оставался отзывчивым даже при больших объёмах данных.
